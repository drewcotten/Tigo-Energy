# Entity Name Reference (Example: `System_Name`)

This reference shows what users see in Home Assistant vs the generated `sensor.xxx` IDs.

- Example system display name: `System_Name`
- Example source display name: `Primary CCA`
- Example array display name: `Array A`
- Example panel labels: `A1`, `A2`, `A3` (sample subset)

## Naming Rules

- Display name (UI): human-friendly, based on system/source/panel labels.
- Entity ID (`sensor.xxx`): slug generated by Home Assistant from suggested object IDs.
- Panel entity IDs intentionally include `system_id` to prevent cross-system collisions when multiple systems all have `A1`, `A2`, etc.

Panel entity ID pattern:
- `sensor.system_<system_id>_panel_<panel_label>_<metric>`

Array entity ID pattern:
- `sensor.system_<system_id>_array_<array_id>_<metric>`

Examples:
- Display name: `System_Name Panel A2 Input power`
- Entity ID: `sensor.system_<system_id>_panel_a2_pin`

Collision note:
- If an ID already exists in your registry, HA appends suffixes (`_2`, `_3`, etc.).
